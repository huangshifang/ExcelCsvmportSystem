## 系统简介

Excel 导入系统是一个基于 Web 的企业级数据导入平台，允许用户将 Excel 文件（.xlsx / .xls）和 CSV 文件（.csv / .tsv / .txt）中的数据导入到 SQL Server 数据库中。系统支持角色权限管理、操作日志追踪、以及 Active Directory 域账号集成。

### 主要功能

- **数据导入**：上传 Excel 或 CSV 文件，自动映射列，批量导入到本地或远程 SQL Server 数据库
- **多服务器支持**：支持注册多个远程 SQL Server 实例，跨服务器导入数据
- **智能列映射**：自动匹配文件列与数据库表列（按名称匹配）
- **实时进度**：大文件导入异步执行，实时显示进度百分比
- **事务支持**：支持原子性导入，失败自动回滚
- **操作日志**：每次导入记录详细信息（用户、文件、行数、状态）
- **仪表板**：实时汇总导入统计指标
- **权限控制**：Admin / Operator / Viewer 三级角色体系
- **AD 集成**：支持 LDAP/Active Directory 域账号登录

### 技术环境

| 项目 | 说明 |
|------|------|
| 浏览器 | Chrome / Edge / Firefox 最新版 |
| 后端服务 | http://localhost:5000 (开发) |
| 前端页面 | http://localhost:5173 (开发) / https://服务器IP (生产) |
| 数据库 | SQL Server |
| 访问协议 | HTTPS（生产环境，自建内部 CA 证书） |

## 快速开始

### 1. 启动后端服务

```bash
cd src
dotnet run
```

后端启动于 `http://localhost:5000`，首次启动会自动创建数据库表和种子数据。

### 2. 启动前端页面

```bash
cd frontend
npm install    # 首次运行需安装依赖
npm run dev
```

前端启动于 `http://localhost:5173`。

### 3. 访问系统

打开浏览器访问 `http://localhost:5173`，使用默认管理员账号登录。

## 登录系统

### 验证码

登录页面需要输入图形验证码（4 位字符）。验证码每次登录都必须填写，点击验证码图片可刷新。登录失败后验证码会自动刷新。

### 本地账号登录

1. 打开系统首页，自动跳转到登录页面
2. 输入用户名和密码
3. 输入验证码（点击验证码图片可刷新）
4. 点击「登录」按钮

**首次登录**：请联系系统管理员获取账号。首次部署时设有默认管理员账号 `admin / admin123`，建议登录后立即修改密码。

### 安全保护

系统对登录接口实施了多重安全防护：
- **验证码必填**：每次登录均需正确输入验证码，防止自动化攻击
- **频率限制**：每个 IP 每分钟最多尝试 10 次登录，超限后返回 503 错误
- **账号锁定**：连续 5 次密码错误后，账号将被锁定 15 分钟

### AD 域账号登录

如果管理员已启用 LDAP 集成，可直接使用域账号（sAMAccountName）和域密码登录。首次登录会自动创建本地账号（Viewer 角色），之后由管理员分配权限。

> **注意**：首次 AD 登录的账号仅有「查看」权限，需联系管理员提升角色。

## 导入数据

> **所需权限**：Import.Execute（Admin、Operator 角色拥有此权限）

### 操作步骤

#### 第一步：配置

1. 点击左侧菜单「导入数据」
2. **选择服务器**（可选）：如果系统配置了多个 SQL Server 实例，先从下拉列表中选择目标服务器。默认为本地服务器
3. **选择数据库**：从下拉列表中选择目标数据库（数据库名称旁标注所属服务器）
4. **选择数据表**：选择要导入的目标表（系统自动读取表的列结构）
5. **上传文件**：点击或拖拽 .xlsx / .xls / .csv / .tsv / .txt 文件到上传区域
6. 配置选项：
   - **包含表头**：文件第一行是否为列名（默认开启）
   - **使用事务**：是否启用事务（开启后全部成功或全部回滚）
   - **批量大小**：每批次插入的行数（可配置 100-10000，默认 1000）
7. 点击「预览并下一步」

#### 第二步：映射列

系统会自动匹配文件列与数据库列（按列名不区分大小写匹配）。

- **已映射列**：显示绿色标签，表示文件列已关联到数据库列
- **未映射列**：需手动从下拉列表中选择对应的数据库列
- **跳过列**：选择「-- 跳过 --」表示不导入该列
- **自增列/主键列**：通常自动跳过，由数据库自动生成

确认所有必要列已正确映射后，点击「执行导入」。

#### 第三步：进度与结果

点击「执行导入」后，系统立即返回任务 ID 并开始后台异步处理：

- **实时进度**：显示圆形进度条和百分比，以及实时处理行数（如 5,000 / 50,000 行）
- **导入成功**：显示导入摘要（总行数、成功行数、失败行数）
- **导入失败**：显示错误信息，可查看详情
- 可以「导入另一个文件」或「查看导入日志」

### 支持的文件格式

- `.xlsx`（Excel 2007+）
- `.xls`（Excel 97-2003）
- `.csv`（逗号分隔值）
- `.tsv` / `.txt`（制表符分隔值）
- Excel 支持多个工作表（自动跳过空工作表，读取第一个有数据的工作表）
- 最大文件大小：200MB

## 服务器实例管理

> **所需权限**：Server.View（查看）、Server.Manage（管理）（仅 Admin 角色拥有此权限）

此功能允许管理员注册多个远程 SQL Server 实例，实现跨服务器的数据导入。系统默认使用本地数据库，无需额外配置。

### 查看服务器列表

1. 点击左侧菜单「服务器实例」
2. 列表显示所有已注册的服务器：名称、描述、状态（活跃/非活跃）、创建时间

### 添加远程服务器

1. 在服务器管理页面，点击「添加服务器」
2. 填写表单：
   - **名称**（必填）：服务器的显示名称
   - **连接字符串**（必填）：SQL Server 连接字符串，格式为 `Server=地址;Database=master;User Id=用户名;Password=密码;TrustServerCertificate=True`
   - **描述**（选填）：备注信息
3. 点击「确认」

> **Docker 部署注意**：如果系统运行在 Docker 容器中，连接宿主机上的 SQL Server 请使用 `host.docker.internal` 作为服务器地址。

### 测试连接

在添加服务器之前，可先测试连接字符串是否有效：

1. 点击「测试连接」按钮
2. 输入连接字符串
3. 点击「测试」，系统会尝试连接并返回 SQL Server 版本信息

### 编辑与删除

- **编辑**：点击服务器的「Edit」按钮，可修改名称、连接字符串、描述、启用/禁用状态
- **删除**：点击「Delete」按钮，确认后删除。如果该服务器还有用户权限关联，需先清除关联后才能删除

### 工作原理

- 添加服务器后，系统自动发现该服务器上的所有可访问数据库
- 导入数据时，从服务器选择器中选择目标服务器
- 远程服务器不可达时，系统会跳过该服务器的数据库列表，不影响本地数据库访问
- 用户权限（数据库/表访问）按服务器独立管理

## 数据库权限管理

> **所需权限**：Database.Manage（仅 Admin 角色拥有此权限）

此功能允许管理员精确控制每个用户能导入哪些数据库中的哪些表。系统支持两种权限粒度：**整库授权**和**逐表授权**。

### 权限粒度

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **整库授权**（通配符） | 勾选数据库的「选择所有表」→ 用户可导入该库所有表 | 用户需要操作整个数据库 |
| **逐表授权** | 展开数据库，仅勾选指定的表 → 用户只能导入已勾选的表 | 限制用户只能操作特定表 |

> **向后兼容**：v1.0 中已授权的数据库权限会自动转为整库授权（通配符模式），无需手动迁移。

### 工作原理

- **管理员**：默认拥有所有数据库和所有表的访问权限，无需额外配置
- **普通用户**：仅能看到被授权的数据库和表，未授权的不会出现在下拉列表中
- **导入验证**：后端在执行导入前会验证用户是否有权访问目标表，即使绕过前端也无法越权

### 为用户配置表级权限

1. 点击左侧菜单「用户管理」
2. 在用户列表中找到目标用户，点击其操作栏的「数据库权限」按钮
3. 弹出对话框中显示按服务器分组的数据库树形选择器：
   - **服务器分组**：数据库按所属服务器分组显示（如「本地」「远程服务器A」）
   - **数据库层级**：每个数据库旁有「选择所有表」复选框，勾选即授予整库访问权限
   - **表层级**：点击数据库左侧箭头展开，显示该库下所有表，可逐表勾选
4. 两种授权方式：
   - **整库授权**：勾选数据库的「选择所有表」，该库所有表均可导入
   - **逐表授权**：展开数据库，仅勾选需要导入的具体表
5. 点击「保存」

> **安全说明**：后端导入服务在 Preview 和 Execute 阶段均会验证表级访问权限，确保数据安全。

## 用户管理

> **所需权限**：User.Manage（仅 Admin 角色拥有此权限）

### 用户列表

1. 点击左侧菜单「用户管理」
2. 列表显示所有用户：ID、用户名、显示名称、邮箱、认证方式、角色、状态

**认证方式标识**：
- `本地`（灰色标签）：使用本地密码登录
- `AD`（蓝色标签）：使用域账号登录

### 添加用户

1. 点击「添加用户」按钮
2. 填写表单：
   - **用户名**（必填）：登录用户名
   - **密码**（选填，至少 6 位）：本地用户的登录密码；AD 用户可留空
   - **显示名称**（必填）：用户的显示名称
   - **邮箱**（选填）
   - **角色**（选填，可多选）：Admin / Operator / Viewer
3. 点击「确认」

### 编辑用户

1. 在用户列表中点击某用户的「Edit」按钮
2. 可修改：显示名称、邮箱、启用/禁用状态
3. 点击「确认」保存

### 删除用户

1. 点击「Delete」按钮
2. 在确认弹窗中点击「Delete」确认删除

> **注意**：默认管理员（ID=1）不可编辑或删除。

### 关联 AD 账号

将一个本地用户关联到 Active Directory 账号：

1. 在用户列表中点击某用户的「Link AD」按钮
2. 在弹出的对话框中输入 AD 用户名进行搜索
3. 在搜索结果中找到目标 AD 用户，点击「Link AD」
4. 关联成功后，该用户即可使用域密码登录

### 重置用户密码（管理员）

管理员可为非 AD 用户重置密码，无需知道旧密码。

1. 在用户列表中点击某用户的「Reset Password」按钮（仅非 AD 用户显示）
2. 输入新密码（至少 6 位）和确认密码
3. 点击「确认」

> **注意**：AD 用户不可通过此功能重置密码，请通过域控管理。

### 解除 AD 关联

1. 在用户列表中点击 AD 用户的「Unlink AD」按钮
2. 确认解除关联
3. 解除后，该用户恢复为本地账号（需设置本地密码才能登录）

## 系统设置

> **所需权限**：System.Manage（仅 Admin 角色拥有此权限）

系统设置页面允许管理员通过 Web 界面配置 LDAP/AD 参数，而无需手动编辑配置文件。配置会持久化到数据库，**修改后即时生效，无需重启服务**。

### 配置 LDAP / AD

1. 点击左侧菜单「系统设置」
2. 在「LDAP / Active Directory 配置」区域填写参数：

| 配置项 | 说明 |
|--------|------|
| **启用 LDAP** | 开关控制是否启用域账号登录 |
| **服务器地址** | AD 域控制器 IP 或域名 |
| **端口** | LDAP 端口，默认 389（SSL 用 636） |
| **SSL** | 是否启用 LDAPS 加密 |
| **域** | 域名（NETBIOS 名） |
| **Base DN** | 搜索基础路径，如 `DC=example,DC=com` |
| **用户过滤器模板** | 默认 `(sAMAccountName={0})` |
| **绑定用户 DN** | 服务账号 DN，用于搜索用户（可选） |
| **绑定密码** | 服务账号密码（留空表示不修改） |

3. 点击「保存配置」

### 测试 LDAP 连接

1. 在「测试 LDAP 连接」区域输入一个域账号的用户名和密码
2. 点击「测试连接」
3. 系统会尝试使用配置的参数进行 LDAP 认证，并返回结果

> **注意**：测试连接会临时保存当前配置到数据库，建议先保存配置再测试。

## LDAP / AD 域集成

### 启用 AD 认证

方式一（推荐）：通过「系统设置」页面配置（见上一节）

方式二：编辑后端配置文件 `src/ExcelImportSystem.API/appsettings.json`：

```json
"Ldap": {
  "Enabled": true,
  "Server": "your-ad-server.example.com",
  "Port": 389,
  "UseSsl": false,
  "Domain": "EXAMPLE",
  "BaseDn": "DC=example,DC=com",
  "UserFilterTemplate": "(sAMAccountName={0})",
  "BindUserDn": "CN=svc,OU=Users,DC=example,DC=com",
  "BindPassword": "your-service-account-password"
}
```

> 配置优先级：数据库中保存的配置 > appsettings.json 文件配置。如果通过系统设置页面保存过配置，将以数据库中的值为准。

### 配置项说明

| 配置项 | 必填 | 说明 |
|--------|------|------|
| `Enabled` | 是 | 设为 `true` 启用 LDAP 认证 |
| `Server` | 是 | AD 域控制器地址（IP 或域名） |
| `Port` | 否 | LDAP 端口，默认 389（SSL 用 636） |
| `UseSsl` | 否 | 是否使用 LDAPS（SSL） |
| `Domain` | 否 | 域名（NETBIOS 名） |
| `BaseDn` | 是 | 搜索基础 DN，如 `DC=example,DC=com` |
| `UserFilterTemplate` | 否 | 用户搜索过滤器，默认 `(sAMAccountName={0})` |
| `BindUserDn` | 否 | 服务账号 DN（用于搜索用户），匿名访问可留空 |
| `BindPassword` | 否 | 服务账号密码 |

### 认证流程

```
用户输入域账号密码
  → 系统尝试本地认证（失败）
  → 使用服务账号绑定 LDAP
  → 搜索用户 DN
  → 使用用户 DN + 密码进行 LDAP 绑定
  → 绑定成功 → 创建/更新本地用户记录 → 生成 JWT → 登录成功
```

### 常用 LDAP 过滤器

| 场景 | 过滤器 |
|------|--------|
| 按 sAMAccountName | `(sAMAccountName={0})` |
| 按 UPN | `(userPrincipalName={0})` |

## HTTPS 安全访问

### 为什么需要安装 CA 证书

系统使用**内部自建 CA** 签发的 SSL 证书，在生产环境中提供 HTTPS 加密访问。由于没有使用公共 CA（如 Let's Encrypt），浏览器首次访问时会提示"连接不安全"或"证书不受信任"。

**这不是安全问题**——只需在每台客户端安装 CA 根证书即可消除警告。

### 客户端安装 CA 证书

> 每台客户端只需安装一次，之后所有内网系统使用同一 CA 签发的证书均可正常访问。

请向系统管理员索取 `ca.crt` 文件，然后按以下步骤安装：

#### Windows

1. 双击 `ca.crt` 文件
2. 点击「安装证书」→ 存储位置选择 **「本地计算机」**
3. 选择「将所有证书放入下列存储」→ 点击「浏览」
4. 选择 **「受信任的根证书颁发机构」** → 确定
5. 下一步 → 完成 → 安全警告弹出时选择 **「是」**

或者在管理员终端执行：
```batch
certutil -addstore Root ca.crt
```

> 企业环境中，管理员可通过**域组策略（GPO）**自动将 CA 证书分发到所有加域计算机。

#### Linux

```bash
sudo cp ca.crt /usr/local/share/ca-certificates/excelimport-ca.crt
sudo update-ca-certificates
```

#### macOS

```bash
sudo security add-trusted-cert -d -r trustRoot \
  -k /Library/Keychains/System.keychain ca.crt
```

### 访问系统

安装 CA 证书后，在浏览器中输入系统地址即可安全访问：

```
https://192.168.1.100    （示例，实际地址请咨询管理员）
```

地址栏应显示锁形图标，表示连接已加密。

## 常见问题

### Q: 忘记管理员密码怎么办？

联系其他管理员通过「用户管理」页面重置密码。如果只有一个管理员且密码已丢失，需要在数据库中手动更新密码哈希：使用 BCrypt 工具生成新密码的哈希值，然后执行 `UPDATE Users SET PasswordHash='新哈希' WHERE Username='admin'`。注意：系统重启不会再重置 admin 密码。

### Q: 导入失败怎么办？

1. 检查 Excel 文件格式是否正确（列名是否与数据库表列匹配）
2. 检查目标表是否存在
3. 查看导入日志中的错误信息
4. 确认数据库连接是否正常

### Q: AD 登录失败？

1. 确认 `Ldap:Enabled` 设为 `true`
2. 检查 AD 服务器地址和端口是否正确
3. 检查 BaseDn 是否正确（可使用 ADSI 编辑器或 `dsquery` 验证）
4. 检查服务账号是否有搜索权限
5. 查看后端日志文件 `logs/excelimport-.log`

### Q: 如何添加新角色？

需要修改后端代码：
1. 在 `Program.cs` 的 `SeedData` 中添加新角色和权限
2. 在 `Program.cs` 的 `AddAuthorization` 中添加新 Policy（如需要）
3. 在前端 `UsersPage.tsx` 的 `roles` 数组中添加新角色

### Q: 如何修改数据库连接？

编辑 `src/ExcelImportSystem.API/appsettings.json` 中的 `ConnectionStrings.DefaultConnection`。

### Q: 用户在导入页面看不到某个数据库或表怎么办？

该用户的数据库/表权限可能未配置。请联系管理员，在「用户管理」中为该用户配置「数据库权限」：
- 如用户需要访问整个数据库，勾选该数据库的「选择所有表」
- 如用户只需导入特定表，展开数据库并勾选对应的表
- **注意**：如果数据库属于远程 SQL Server 实例，管理员也必须在权限配置中展开该远程实例下的数据库为其授权，系统会自动区分不同实例上的同名数据库

### Q: 如何不重启服务就修改 LDAP 配置？

管理员可通过「系统设置」页面修改 LDAP 配置，保存后即时生效，无需重启服务或修改配置文件。

### Q: 登录提示 "Account is locked" 怎么办？

连续 5 次输入错误密码后，账号会被锁定 15 分钟。请等待锁定时间过后再试，或联系管理员重置密码（重置密码后锁定自动解除）。

### Q: 登录提示 "Invalid captcha code" 怎么办？

验证码错误或已过期。请重新输入当前显示的验证码，或点击验证码图片刷新后重试。

### Q: AD 用户可以修改密码吗？

AD 用户的密码由域控制器管理，不能在系统中修改。如需修改 AD 密码，请在域控上操作或使用 Windows 标准密码修改流程。

### Q: 系统支持哪些数据库？

当前仅支持 SQL Server。如需支持其他数据库：
1. 更换 EF Core Provider（如 Npgsql for PostgreSQL）
2. 修改 `TableService.cs` 中的 `INFORMATION_SCHEMA` 查询（各数据库的系统视图有差异）

### Q: 浏览器提示"连接不安全"怎么办？

这是因为系统使用了内部 CA 签发的证书。请参考 [HTTPS 安全访问](#https-安全访问) 章节安装 CA 根证书，安装后警告即消除。

### Q: 安装 CA 证书安全吗？

安全的。内部 CA 只能为内网系统签发证书，不会被外部网站利用。CA 私钥由管理员保管，不在客户端安装。

### Q: 如何将 CA 证书分发给所有客户端？

- 小型部署（< 10 台）：将 `ca.crt` 文件通过共享文件夹或邮件发送给用户，用户按步骤安装
- 企业环境：通过 Active Directory 组策略（GPO）自动推送，无需用户操作
- 管理员参考：`offline-deploy/README.md` 中的「HTTPS 证书说明」章节

### Q: 如何添加远程 SQL Server 进行跨服务器导入？

管理员可通过「服务器实例」页面注册远程 SQL Server：
1. 点击左侧菜单「服务器实例」
2. 点击「添加服务器」，填写名称和连接字符串
3. 可使用「测试连接」功能验证连接是否可用
4. 添加后，导入页面的服务器选择器中即可看到新服务器

### Q: Docker 容器中如何连接宿主机上的 SQL Server？

连接字符串中的服务器地址使用 `host.docker.internal` 即可。示例：
`Server=host.docker.internal;Database=master;User Id=sa;Password=yourpassword;TrustServerCertificate=True`

### Q: 远程服务器不可达会影响系统使用吗？

不会。系统在发现远程服务器上的数据库时会捕获连接异常并记录日志，不影响本地数据库的访问和导入。不可达的远程服务器对应的数据库列表不会出现在下拉列表中，但本地数据库正常可用。

### Q: Excel 文件有多个工作表，系统读取哪个？

系统会自动跳过空的占位工作表（仅含一个空白 A1 单元格），读取第一个包含实际数据的工作表。例如，文件有 sheet1（空）和 sheet2（数据），系统将自动读取 sheet2。如果你需要导入特定工作表，请确保目标数据在第一个非空工作表中，或删除不需要的空工作表。

### Q: CSV 文件没有表头行怎么办？

在上传文件时将「包含表头」开关关闭即可。系统会自动生成 `Column1`、`Column2` 等列名，并将第一行作为数据读取。
